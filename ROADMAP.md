# План развития

## Направление

Этот форк развивает FlClash как Android-first VPN-клиент с упором на приватность, предсказуемые Android-релизы и собственную продуктовую линию без опоры на кроссплатформенный маркетинг апстрима.

Ближайшая цель не в том, чтобы обещать нереалистичное «полное отсутствие детекта». Цель в том, чтобы убрать устранимые клиентские утечки, сделать безопасные настройки дефолтом, держать репозиторий в дисциплине и подготовить площадку для следующих Android-сетевых функций.

## Текущий трек

### 1. Усиление приватности Android

- Держать localhost listener surface закрытым в Android VPN-режиме.
- Сокращать легко узнаваемые статические значения там, где это безопасно.
- Расширять регрессионное покрытие для чувствительных путей генерации конфигурации и обновления runtime-настроек.

### 2. Android-only релизы

- Собирать только Android-артефакты из GitHub Actions этого форка.
- Держать release notes, шаблоны и технические заметки внутри репозитория.
- Убирать из публичной поверхности обещания и процессы, которые относятся к legacy desktop-слою.

### 3. Основа для новых функций

- Подготовить архитектурную основу для встроенного ByeByeDPI и родственных Android-функций обхода блокировок.
- Развивать profile-driven Android-функции, которые удобнее поддерживать через конфиг, чем через ручные переключатели.
- Предпочитать поведение «безопасно по умолчанию» для Android-ориентированных сценариев.

### 4. Дисциплина репозитория

- Держать `main` читаемой: один логический commit на одно изменение без цепочек `fix`, `revert`, `polish release notes`.
- Не разносить релизные правки, changelog и код по множеству шумных коммитов.
- Сопровождать архитектурные решения короткой документацией, если они меняют публичную модель продукта.

## Что не является целью

- Заявлять, что клиентское приложение способно полностью скрыть VPN от публичных Android API без root.
- Подменять device-side root/Xposed masking для пользователей, которым нужна более сильная защита от детекта.
- Поддерживать Windows, macOS и Linux как актуальные платформы продукта только потому, что такой код исторически унаследован от апстрима.
