# Политика сопровождения

## Scope

Этот форк ведётся как Android-first продукт. Поддерживаемый публичный контур включает:

- Android-код, Android-релизы и Android-документацию;
- изменения для приватности, обновления, профилей и сетевого поведения на Android;
- подготовку к следующим Android-функциям, включая встроенный ByeByeDPI.

Унаследованный desktop-код может оставаться в дереве как legacy-слой апстрима, но он не считается продуктовой поверхностью форка.

## Правила для коммитов

- Один логический commit должен закрывать одну понятную задачу.
- Не отправляйте в `main` цепочки вида `fix`, `fix fix`, `revert`, `re-apply`, `polish release notes`.
- Если изменение требует нескольких промежуточных шагов в рабочей ветке, перед публикацией схлопывайте их в осмысленные commits.
- Документацию, changelog и код, относящиеся к одному изменению, лучше держать в одном commit или в короткой предсказуемой серии.
- Отдельные релизные commits допустимы только когда они действительно меняют версию, теговый пакет или release notes для конкретного релиза.

## Правила для веток и релизов

- `main` остаётся основной веткой продукта.
- Теги `v*` публикуют стабильные Android-релизы.
- Теги `v*-preN` публикуют Android prerelease.
- Перед тегом проверьте, что верхняя секция `CHANGELOG.md` уже готова и не содержит служебного текста.

## Что помогает держать репозиторий чистым

- Удаляйте или переписывайте промежуточные WIP-коммиты до публикации ветки.
- Не делайте отдельные шумовые commits только ради переформулировки release notes, если это можно включить в целевой commit.
- Если изменение вводит новую Android-модель поведения, обновляйте документацию в том же цикле работы.
- Если в коде остаётся legacy-слой апстрима, явно отмечайте в документации, поддерживается он или нет.
