# Политика безопасности

## Область фокуса

Этот форк рассматривает безопасность через Android-only призму. В первую очередь нас интересуют:

- доступные другим приложениям localhost listeners и прокси;
- неаутентифицированные локальные API, через которые можно получить метаинформацию о VPN;
- рискованные Android-дефолты в `VpnService` и связанных конфигурационных путях;
- ошибки релизного контура, из-за которых пользователю может прийти неподписанный, несоответствующий или не из того дерева собранный Android-артефакт.

## О чём стоит сообщать

Открывайте security-обращение, если нашли:

- listener на localhost, который остаётся доступным в Android VPN-режиме;
- `external-controller`, системный proxy или другой локальный канал, доступный соседнему приложению;
- путь обновления или импорта профиля, который возвращает небезопасные Android-дефолты;
- регрессию в Android split tunneling, из-за которой трафик выходит за ожидаемые границы;
- проблему в релизном процессе, которая позволяет опубликовать не тот Android-бинарник.

## Что не решается только клиентом

Этот форк не может полностью скрыть наличие Android VPN от публичных Android API без device-side root hooking.

Примеры сигналов, которые приложение всё ещё может увидеть:

- `NetworkCapabilities.TRANSPORT_VPN`;
- отсутствие `NET_CAPABILITY_NOT_VPN`;
- часть route, interface, DNS, MTU и других TUN-сигналов.

Фон, границы модели угроз и текущая стратегия смягчения описаны в [docs/android-vpn-hardening.md](docs/android-vpn-hardening.md).
