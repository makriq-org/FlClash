# v0.9.01-pre5

## Что изменили технически

### Android split tunneling: маски, regex и исключения

- Android split tunneling по пакетам теперь понимает не только точные package names, но и маски вроде `*.yandex.*`, регулярные выражения через `re:` и исключения через `!`.
- Правила можно свободно смешивать в одном `tun.include-package` или `tun.exclude-package`, поэтому профили стали компактнее и лучше описывают семейства приложений.
- Сценарий "все приложения Яндекса напрямую, кроме Яндекс Браузера" теперь задаётся одной последовательностью правил без ручного перечисления каждого пакета.

### Android split tunneling: предсказуемое раскрытие правил

- Маски, regex и negation-правила раскрываются на стороне FlClash по списку реально установленных Android-приложений ещё до старта `VpnService`, поэтому runtime Android по-прежнему получает только точные package names.
- Порядок правил теперь сохраняется и для inline YAML, и для списков из `*-package-file` / `*-package-url`, так что последнее совпавшее правило всегда побеждает.
- Несовпавшие правила больше не ломают запуск профиля: они аккуратно игнорируются с предупреждением в лог, чтобы один и тот же профиль можно было использовать на разных устройствах.

### Верификация

- Регрессионные тесты дополняются кейсами на glob, `re:`, `!`-исключения и сохранение порядка правил в файловых списках.
- GitHub Actions branch pipeline проверяет `flutter analyze`, `flutter test test/common` и Android branch build для ветки с `pre5`.
- GitHub Actions release pipeline публикует prerelease `v0.9.01-pre5` после успешной сборки матрицы платформ.
