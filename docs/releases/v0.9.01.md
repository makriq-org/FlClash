# v0.9.01

## Контекст

Этот стабильный релиз собирает в одну поддерживаемую линию всю серию `v0.9.01-pre*`. Основной фокус релиза — довести profile-driven Android split tunneling до рабочего стабильного состояния, закрыть связанные с ним риски безопасности и выровнять поведение встроенного обновления приложения.

## Что изменили технически

### Android split tunneling через профиль

- Android split tunneling теперь можно задавать прямо в профиле через `tun.exclude-package` и `tun.include-package`.
- Поддержаны точные package names, списки из файлов и URL, а также profile-managed режимы include/exclude без ручной настройки на устройстве.
- Для правил по приложениям добавлены маски, регулярные выражения через `re:` и исключения через `!`, чтобы один профиль мог описывать семейства приложений, а не только отдельные пакеты.
- Раскрытие масок и regex выполняется на стороне FlClash по списку реально установленных приложений, а в Android runtime по-прежнему передаются только точные package names.
- При отсутствии доступа к списку приложений профиль больше не падает: exact-правила продолжают работать, а правила, которым нужен installed-app inventory, пропускаются мягко и предсказуемо.

### Безопасность и границы профиля

- Пути к `include-package-file` и `exclude-package-file` жёстко ограничены каталогом профилей FlClash и контролируемым cache-path для удалённых списков.
- Android `FilesProvider` больше не принимает произвольные absolute paths: `document id` привязан к внутреннему data-каталогу приложения и не позволяет выйти за его пределы.
- HTTPS-исключение для самоподписанных сертификатов сохранено только для loopback-хостов, а не для произвольных адресов.

### Обновление приложения и релизный контур

- Встроенный Android updater перед запуском installer теперь останавливает активный VPN, чтобы обновление не зависало на открытом `VpnService`.
- Исправлен импорт `LogLevel` в Android update dialog, из-за которого prerelease-сборка падала на этапе `flutter analyze`.
- В релизную документацию и release notes внесено разделение на короткий пользовательский слой и отдельный технический changelog.

## Совместимость и риски

- Старые сценарии, опиравшиеся на произвольные absolute paths вне каталога профилей, теперь намеренно блокируются как небезопасные.
- Профили, использующие маски и regex для правил по приложениям, дают полный эффект только там, где оболочка позволяет получить список установленных приложений; при отказе доступ не ломает запуск профиля, но снижает точность таких правил.

## Проверка

- Для серии `v0.9.01` добавлены и обновлены регрессионные тесты на include/exclude-правила, file/url-backed package lists, glob/regex/negation-сценарии и попытки выхода за границы каталога профилей.
- Перед публикацией стабильного релиза нужно прогнать `flutter analyze`, `flutter test test/common` и полную release-сборку по матрице поддерживаемых платформ.
