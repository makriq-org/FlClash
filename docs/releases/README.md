# Технические changelog-файлы

В этой папке лежат подробные технические заметки по конкретным релизам.

- Имя файла должно совпадать с тегом релиза: `vX.Y.Z.md` или `vX.Y.Z-preN.md`.
- `CHANGELOG.md` остаётся коротким пользовательским слоем без CI, тестовых команд и внутренних идентификаторов.
- Если файл для тега существует, GitHub Release автоматически получает ссылку на него при публикации.

Обычно в техническом changelog достаточно таких разделов:

- что изменили технически;
- какие ограничения или риски закрыли;
- какие миграции или совместимость нужно учитывать;
- как это проверяли.

За основу можно брать шаблон из [docs/releases/_template.md](_template.md).
