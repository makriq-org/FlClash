# v0.8.93

## Что изменили технически

### Защита Android VPN

- Добавлен Android-only слой защиты, который переписывает сгенерированную Clash runtime-конфигурацию в VPN-режиме.
- Все локальные inbound listener-порты в Android VPN-режиме теперь принудительно закрываются:
  - `port=0`
  - `socks-port=0`
  - `mixed-port=0`
  - `redir-port=0`
  - `tproxy-port=0`
- В усиленном Android VPN-режиме `external-controller=''` теперь выставляется принудительно, а `allow-lan=false` удерживается постоянно.
- Собственные Android-запросы приложения больше не завязаны на localhost mixed proxy при активном VPN.

### Снижение распознаваемости

- IPv4 и IPv6 адреса Android-туннеля теперь рандомизируются при каждом запуске VPN.
- Статичный `VpnService` session label заменён на нейтральный `VPN`.

### Верификация и автоматизация релизов

- Добавлены регрессионные тесты для защиты Android VPN.
- Добавлен отдельный веточный Android workflow сборки для GitHub Actions.
- Основной workflow релиза обновлён так, чтобы этот форк мог собираться и публиковаться из собственного репозиторного контекста.
- Настройка подписи Android сделана условной по секретам репозитория, чтобы сопровождение форка было проще.

### Независимость репозитория

- Основной README переписан вокруг целей этого форка по усилению приватности.
- В репозиторий добавлены исследование защиты Android VPN и документация по безопасности.
- Метаданные релизов и шаблоны обновлены так, чтобы форк больше не зависел от апстримного брендинга и инфраструктурных предположений.
