# v0.9.02-pre1

## Контекст

Этот bugfix-предрелиз заменяет первый ошибочный `v0.9.02-pre1`. После восстановления raw Android split tunneling из profile YAML мы снова начали видеть два класса проблем:

- profile-managed `include-package-file` / `exclude-package-url` могли теряться после разбора через Clash core;
- при отсутствии надёжного inventory установленных приложений внешние package lists могли засорять итоговый Android config чужими exact package names;
- во время сборки runtime-конфига профильные `tun.route-address` и `tun.auto-route` перетирались app-side patch-настройками.

## Что изменили технически

- Android profile access control теперь восстанавливает `include-package`, `exclude-package`, `include-package-file`, `exclude-package-file`, `include-package-url` и `exclude-package-url` из исходного YAML профиля до normalizing/runtime-шага.
- Exact package selectors, пришедшие из внешних `file/url`-списков, больше не попадают в итоговый Android override без актуального inventory установленных приложений.
- Inline exact selectors, заданные прямо в самом профиле, сохраняют прежнее мягкое поведение и могут продолжать работать даже без inventory.
- `makeRealProfileTask(...)` больше не перезаписывает профильные `tun.route-address` и `tun.auto-route`, если они уже заданы в самом профиле.
- Обновлён `pubspec.yaml`, чтобы проект снова резолвился с актуальным Flutter SDK из рабочего `nix`-окружения.

## Совместимость и риски

- Изменение intentionally делает внешний split tunneling fail-closed: если inventory приложений временно недоступен, внешний список больше не будет “угадываться” по exact package names.
- Профили, которые полагались на external lists без доступа к package inventory, теперь безопаснее, но могут временно не применять split tunneling до получения inventory.
- Профильные маршруты TUN снова становятся source of truth, кроме тех случаев, где runtime явно переопределяет другие части Android hardening.

## Проверка

- Добавлены регрессионные тесты на:
  - восстановление `exclude-package-url` из raw profile YAML;
  - игнорирование external exact selectors без inventory приложений;
  - сохранение профильного `tun.route-address` / `tun.auto-route` в итоговом config.
- Локально через `nix` выполнены `dart format`, `flutter pub get`, `flutter analyze lib test setup.dart` и `flutter test test/common/android_vpn_hardening_test.dart`.
