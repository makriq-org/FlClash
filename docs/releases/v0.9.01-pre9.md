# v0.9.01-pre9

## Контекст

Этот предрелиз завершает очередную волну усиления Android split tunneling и встроенного обновления приложения. Целью было закрыть оставшиеся пути выхода за границы профиля, убрать лишние исключения в сетевом контуре и стабилизировать установку обновления поверх активного VPN.

## Что изменили технически

- Пути к `include-package-file` и `exclude-package-file` жёстко ограничены каталогом профилей FlClash, включая явные cache-path для удалённых списков пакетов.
- Android `FilesProvider` больше не принимает произвольные absolute paths: `document id` теперь привязан к внутреннему data-каталогу приложения и отсекает выход за его пределы.
- Встроенный Android updater перед запуском installer теперь сначала останавливает активный VPN, чтобы обновление не зависало на открытом `VpnService`.
- HTTPS-исключение для самоподписанных сертификатов сохранено только для loopback-хостов, а не для любых адресов.
- Исправлен импорт `LogLevel` в Android update dialog, из-за которого prerelease-сборка падала на этапе `flutter analyze`.

## Совместимость и риски

- Подход с ограничением путей оставляет рабочими только списки приложений внутри ожидаемого каталога профилей и его управляемого cache-path.
- Старые сценарии, опиравшиеся на произвольные absolute paths вне профиля, теперь намеренно блокируются как небезопасные.

## Проверка

- Регрессионные тесты дополнены кейсами на выход за пределы каталога профилей для локальных и удалённых package lists.
- Перед публикацией этого предрелиза нужно прогнать `flutter analyze`, `flutter test test/common` и полную release-сборку по матрице платформ.
