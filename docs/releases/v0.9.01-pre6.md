# v0.9.01-pre6

## Что изменили технически

### Android split tunneling: мягкий отказ от списка приложений

- Если доступ к списку установленных приложений не получен, профиль больше не падает с ошибкой: FlClash продолжает стартовать как обычно.
- В таком режиме точные package names продолжают работать, а маски и regex аккуратно пропускаются до тех пор, пока пользователь не разрешит доступ.
- Это делает profile-driven split tunneling безопаснее для первого запуска и убирает неприятный hard failure при отказе от доступа.

### Android split tunneling: объяснение перед доступом к приложениям

- Перед первым обращением к списку установленных приложений FlClash теперь показывает короткое пояснение, зачем это нужно для package masks и поддержки российских приложений, которые могут не пускать при активном VPN.
- В том же окне пользователь может отказаться, и это решение будет уважаться без повторных навязчивых запросов.

### Верификация

- Регрессионные тесты обновлены под graceful fallback без списка приложений и продолжают покрывать exact/glob/regex/negation-сценарии.
- GitHub Actions branch pipeline проверяет `flutter analyze` и `flutter test test/common` для `pre6`; после этого можно не дожидаться конца длинной release-матрицы.
